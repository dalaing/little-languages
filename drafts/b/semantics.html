Values

A subset of our terms will be values.

A value is a term that cannot be evaluated any further - that’s what it means to be a value.

We could write a test for whether a term is a value:

isValue :: Term 
        -> Bool
isValue TmTrue  = 
  True
isValue TmFalse =
  True
isValue _       = 
  False

We’re going to take a different path which will simplify the testing and the composability of our system. It’s may look like a lot of unnecessary complexity, but the reasons for these changes will soon be made clear.

We define the function value:

value :: Term 
      -> Maybe Term
value TmTrue  =
  Just TmTrue
value TmFalse = 
  Just TmFalse
value _       =
  Nothing

which returns Nothing for non-values, from which we can define isValue:

isValue :: Term 
        -> Bool
isValue = 
  isJust .
  value

We can break value up, so that we have a function for each of the rules. The functions will use Maybe, to handle the case where the rule doesn’t hold.

So

becomes

valueTmTrue :: Term 
            -> Maybe Term
valueTmTrue TmTrue = 
  Just TmTrue
valueTmTrue _ =
  Nothing

and

becomes

valueTmFalse :: Term 
             -> Maybe Term
valueTmFalse TmFalse = 
  Just TmFalse
valueTmFalse _ =
  Nothing

Now we need to combine them back together, which we can do with the Alternative instance for Maybe:

value :: Term 
      -> Maybe Term
value tm =
  valueTmTrue tm <|>
  valueTmFalse tm

This can be generalized to a list of rules, by using asum from Data.Foldable:

asum :: (Foldable t, Alternative f) 
     => t (f a) 
     -> f a

to get:

value :: Term 
      -> Maybe Term
value tm =
  asum 
    [ valueTmTrue tm
    , valueTmFalse tm
    ]

We can simplify that:

value :: Term 
      -> Maybe Term
value tm =
  asum . 
  map ($ tm) $
    [ valueTmTrue
    , valueTmFalse
    ]

and then we can separate out the rules from the value function:

valueRules :: [Term -> Maybe Term]
valueRules = 
  [ valueTmTrue
  , valueTmFalse
  ]

value :: Term 
      -> Maybe Term
value tm =
  asum . 
  map ($ tm) $
  valueRules

becomes:

eIfTrue :: Term 
        -> Maybe Term
eIfTrue (TmIf TmTrue t2 _) = 
  Just t2
eIfTrue _                  =
  Nothing

and

becomes:

eIfFalse :: Term 
         -> Maybe Term
eIfFalse (TmIf TmFalse _ t3) = 
  Just t3
eIfFalse _                   = 
  Nothing

The last case is recursive:

and becomes:

eIf :: (Term -> Maybe Term) 
    -> Term 
    -> Maybe Term
eIf step (TmIf t1 t2 t3) = do
  t1' <- step t1
  return $ TmIf t1' t2 t3
eIf _ _                  = 
  Nothing

We pass the step function into the rule so that we can worry about the recursion later.

The step function works in the Maybe monad since it isn’t defined for all terms. In the case of B it isn’t defined for values, but we’ll push that further later on.

In the general case for our natural deduction rules, we’ll be working in a monad with a notion of failure, and this failure will encompass the pattern match failures for our rules.

We can now combine these rules into a smallStep function, in the same way that we combined the rules for values into the value function.

smallStepRules :: [Term -> Maybe Term]
smallStepRules = 
  [ eIfFalse
  , eIfTrue
  , eIf smallStep
  ]

smallStep :: Term 
          -> Maybe Term
smallStep tm =
  asum .
  map ($ tm) $
  smallStepRules

The smallStepRules and smallStep function are mutually recursive. The extra level of indirection allows us to define rules that use recursion in a different module to smallStep.

This means we can define more rules elsewhere, and add them to smallStepRules when we think they’re ready for use.

Theorems

We still could have messed up our rules. The patterns could overlap, we could have forgotten something, or there could be a typo in our code. We want to try to rule that out.

We do this by coming up with behaviours and properties that we think should be reasonably, and then demonstrating that they are true.

In the PLT space, this is done by coming up with theorems and proving them. Many of these theorems are phrased in such a way that they will hold across a useful subset of languages. We can then prove those theorems hold in our language of interest, often using proof techniques that are made familiar to us when we learn and start using those theorems.

For now we have a few properties of interest.

  • determinism
  • we get this for free from the fact that small-step is a pure function
  • partly a quirk of how we are combining the rules
  • for all terms, exactly one of the combination of the value rules and the small-step rules apply

As a bonus, all of these rules reduce the size of terms they act on. Since we have covered all of the cases, and you can only reduce the size of a finite term so far, we know that all finite terms will eventually evaluate to a value.
